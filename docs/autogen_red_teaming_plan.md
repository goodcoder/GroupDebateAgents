# AutoGen Adversarial Red Teaming Implementation Plan

## Goal
Implement a multi-agent debate system using Microsoft AutoGen focused on Adversarial Red Teaming. The system will leverage conversational dynamics and aggressive system prompting to stress-test ideas and plans generated by a Primary Agent without relying on complex external search algorithms like Monte Carlo Tree Search (MCTS).

## 1. System Architecture
We will utilize the core components of the AutoGen framework to facilitate a structured debate:
*   `ConversableAgent`: The building block for our distinct personas.
*   `GroupChat`: The environment where the agents interact.
*   `GroupChatManager`: The orchestrator that controls turn-taking and broadcasts messages.

## 2. Agent Personas
The system requires three distinct agents to create a meaningful adversarial dynamic.

### A. The Planner (Primary Agent)
*   **Role**: To generate the initial comprehensive plan or solution based on the user's prompt.
*   **System Prompt Focus**: "You are a strategic planner. Your goal is to construct a detailed, logical, and effective plan to solve the given problem. You must defend your reasoning but also be willing to revise your plan if genuine, critical flaws are exposed."

### B. The Adversary (Red Team Critic)
*   **Role**: To aggressively attack the Planner's proposal, looking for edge cases, logical fallacies, security risks, or impracticalities.
*   **System Prompt Focus**: "You are an adversarial red-team reviewer. Your ONLY goal is to find critical flaws, dangerous assumptions, and edge cases in the Planner's proposal. Do not be helpful or suggest solutions. Ask aggressive 'What if...' questions and explicitly state why the plan will fail in the real world."

### C. The Judge (Moderator)
*   **Role**: To monitor the debate, extract the final refined plan, and terminate the loop.
*   **System Prompt Focus**: "You are the impartial Judge. Monitor the debate between the Planner and the Adversary. Once the Planner has successfully addressed the Adversary's major concerns, or if the conversation starts going in circles (Degeneration-of-Thought), summarize the final, hardened plan and output the exact word 'TERMINATE' to end the session."

## 3. Interaction Flow
1.  **Initiation**: The User Proxy agent initiates the chat by providing a challenging topic or scenario to the `GroupChatManager`.
2.  **Round 1**: The Manager calls on the **Planner** to draft the initial architecture/plan.
3.  **Round 2**: The Manager calls on the **Adversary** to review and attack the plan.
4.  **Refinement Loop**: The Planner revises or defends its position. The Adversary attacks again. This loops for a defined number of `max_rounds` (e.g., 4-6).
5.  **Termination**: The **Judge** observes the progress. Once a consensus is forced or the maximum rounds are reached, the Judge synthesizes the final robust output and ends the debate.

## 4. Next Steps
1.  **Environment Setup**: Ensure AutoGen (`pyautogen`) is installed in the current virtual environment.
2.  **Implementation**: Create the Python script (`autogen_debate.py`) instantiating the agents and configuring the `GroupChat`.
3.  **Testing**: Run the script with a sample complex topic (e.g., "Design a secure architecture for a decentralized voting app") to observe the adversarial dynamic in action.
